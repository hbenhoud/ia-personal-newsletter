# AI Personal Newsletter

A self-hosted Go CLI that automatically generates a personal AI-focused newsletter. It fetches RSS feeds, filters articles by semantic similarity to your profile using free embedding APIs, summarizes them with a free LLM, and renders a static HTML site you can open locally or deploy to GitHub Pages.

## Prerequisites

- **Go 1.22+** — [install](https://go.dev/dl/)
- At least one free API key:
  - **Groq** (LLM, free tier) — [console.groq.com](https://console.groq.com)
  - **Gemini** (LLM + embeddings, free tier) — [aistudio.google.com](https://aistudio.google.com)
  - **HuggingFace** (embeddings, free tier) — [huggingface.co/settings/tokens](https://huggingface.co/settings/tokens)

Set environment variables before running:

```bash
export GROQ_API_KEY=gsk_...
export GEMINI_API_KEY=AIza...
# or for HuggingFace embeddings:
export HUGGINGFACE_API_KEY=hf_...
```

## Installation

```bash
git clone https://github.com/hbenhoud/ia-personal-newsletter.git
cd ia-personal-newsletter
go build -o newsletter ./cmd/newsletter
```

This produces a `newsletter` binary in the current directory. Optionally move it to your `$PATH`:

```bash
mv newsletter /usr/local/bin/newsletter
```

## Configuration

All configuration lives in the `config/` directory.

### newsletter.yaml

Controls sources, filtering, embedding, LLM, and output settings.

```yaml
sources:
  rss:
    - https://huggingface.co/blog/feed.xml
    - https://arxiv.org/rss/cs.AI
    - https://hnrss.org/frontpage
    # Add any RSS/Atom feed URL here

filtering:
  mode: semantic              # semantic | keyword | both
  similarity_threshold: 0.72  # 0.0–1.0; lower = more articles, higher = stricter
  max_articles: 20            # maximum articles passed to LLM summarization
  exclude_keywords: []        # hard exclusions applied before embedding (e.g. [crypto, NFT])

embedding:
  provider: gemini            # gemini | huggingface
  model: text-embedding-004   # Gemini model; or e.g. sentence-transformers/all-MiniLM-L6-v2 for HF
  api_key_env: GEMINI_API_KEY # name of the env var holding the API key
  cache_path: ./.cache/embeddings.json

llm:
  provider: groq              # groq | gemini
  model: llama-3.3-70b-versatile
  api_key_env: GROQ_API_KEY

output:
  site_dir: ./output
  items_per_issue: 10         # final number of articles rendered in the newsletter
```

### profile.md

Generated by `newsletter profile setup`. You can also edit it directly.

```markdown
# User profile

I am an expert-level backend developer. I am interested in open source LLMs,
RAG, autonomous agents, local inference tooling, and NLP research papers.

## Topics to avoid
crypto, business, fundraising, AI art

## Preferences
language: en          # en | fr — language for the newsletter output
recency_days: 7       # only articles published in the last N days
theme: dark           # minimal | dark | paper | terminal
level: expert         # beginner | intermediate | expert
```

**How the profile is used:**
The full text of `profile.md` is embedded into a vector that serves as the reference point. Every article is also embedded, and only articles whose vector is close enough (above `similarity_threshold`) are kept. The `Interests` and `Level` fields are also injected into the LLM summarization prompt to personalise the output.

## Architecture

```
ia-personal-newsletter/
├── cmd/newsletter/main.go         CLI entry point — parses args, wires components
├── internal/
│   ├── config/config.go           Loads newsletter.yaml + profile.md, runs wizard
│   ├── ingestion/rss.go           Concurrent RSS fetch, Article normalisation, URL dedup
│   ├── filtering/semantic.go      3-pass filter: recency → exclusion → cosine similarity
│   ├── embedding/
│   │   ├── embedder.go            Embedder interface
│   │   ├── gemini.go              Gemini text-embedding-004 implementation
│   │   ├── huggingface.go         HuggingFace Inference API implementation
│   │   ├── cache.go               SHA256-keyed JSON cache (load on start, flush on exit)
│   │   └── factory.go             Creates the configured Embedder
│   ├── llm/
│   │   ├── provider.go            Provider interface + GenerationConfig
│   │   ├── groq.go                Groq (OpenAI-compatible) implementation
│   │   ├── gemini.go              Gemini generateContent implementation
│   │   └── factory.go             Creates the configured Provider
│   ├── generation/summarizer.go   Renders prompt template, calls LLM, parses TL;DR output
│   └── site/generator.go          Writes output/YYYY-WW/index.html + output/index.html
├── config/
│   ├── newsletter.yaml            User-editable configuration
│   └── profile.md                 Generated by wizard
├── templates/
│   ├── newsletter.html            Issue page template
│   ├── index.html                 Issues list page template
│   ├── prompts/summarize.tmpl     LLM prompt template
│   └── themes/
│       ├── minimal.css
│       ├── dark.css
│       ├── paper.css
│       └── terminal.css
├── .cache/embeddings.json         Persistent embedding cache
└── output/                        Generated static site
    ├── index.html
    └── YYYY-WW/index.html
```

## Execution Pipeline

```
newsletter generate
       │
       ▼
[1] Load config
    config/newsletter.yaml + config/profile.md
       │
       ▼
[2] Ingest RSS feeds (concurrent)
    github.com/mmcdole/gofeed
    → []Article{Title, URL, Content, Source, Published}
       │
       ▼
[3] Pre-filter (no API calls)
    ├─ Drop articles older than recency_days
    ├─ Drop articles matching exclude_keywords
    └─ Deduplicate by URL
       │
       ▼
[4] Embed articles + profile
    ├─ Load embedding cache (.cache/embeddings.json)
    ├─ Embed profile.md text → reference vector
    └─ Embed each article (title + description, capped at 2000 chars)
       │  (cached vectors skip API call)
       ▼
[5] Score + rank
    ├─ Cosine similarity(article_vec, profile_vec) for each article
    ├─ Keep articles above similarity_threshold
    └─ Sort descending, keep top max_articles
       │
       ▼
[6] Summarize (LLM)
    For each selected article:
    └─ Render summarize.tmpl → call LLM → parse TL;DR + Why it matters
       │
       ▼
[7] Render static site
    ├─ Load theme CSS (templates/themes/{theme}.css)
    ├─ Render newsletter.html → output/YYYY-WW/index.html
    └─ Regenerate output/index.html (issues list)
       │
       ▼
    Done. Open output/YYYY-WW/index.html
```

## Available Themes

Set via `newsletter profile setup` or by editing `theme:` in `config/profile.md`.

| Theme | Style | Best for |
|---|---|---|
| `minimal` | White background, system sans-serif font, clean layout | Everyday reading, light mode |
| `dark` | Dark (#0d1117) background, blue/purple accents | Night reading, GitHub-like feel |
| `paper` | Cream background, Georgia serif, print-like | Long-form reading, distraction-free |
| `terminal` | Black background, monospace font, green accents | Hacker aesthetic, terminal lovers |

All themes:
- Embed CSS inline — no internet required to view
- Use max-width 680px single-column layout
- Display the similarity score discreetly per article

## Adding a New Provider

### New LLM Provider

1. Create `internal/llm/myprovider.go` implementing the `Provider` interface:

```go
type Provider interface {
    Complete(ctx context.Context, prompt string, cfg GenerationConfig) (string, error)
    Name() string
}
```

2. Register it in `internal/llm/factory.go`:

```go
case "myprovider":
    return NewMyProvider(apiKey, model), nil
```

3. Use it: set `llm.provider: myprovider` and `llm.api_key_env: MY_API_KEY` in `newsletter.yaml`.

### New Embedding Provider

1. Create `internal/embedding/myprovider.go` implementing the `Embedder` interface:

```go
type Embedder interface {
    Embed(ctx context.Context, text string) ([]float64, error)
    EmbedBatch(ctx context.Context, texts []string) ([][]float64, error)
}
```

2. Register it in `internal/embedding/factory.go`:

```go
case "myprovider":
    return NewMyEmbedder(apiKey, model, cache), cache, nil
```

3. Use it: set `embedding.provider: myprovider` and `embedding.api_key_env: MY_EMBED_KEY` in `newsletter.yaml`.

The cache is automatically shared — no changes needed to `cache.go`.

## Deploying the Site

The `output/` directory is a self-contained static site. Each HTML file embeds all CSS inline.

### GitHub Pages

```bash
# Push the output directory to a gh-pages branch
git subtree push --prefix output origin gh-pages
```

Or use a GitHub Actions workflow to run `newsletter generate` on a schedule and push the output.

### Netlify Drop

Drag and drop the `output/` folder at [app.netlify.com/drop](https://app.netlify.com/drop).

### Local preview

```bash
open output/$(ls output | grep -v index | sort -r | head -1)/index.html
# or serve it:
cd output && python3 -m http.server 8080
```

## FAQ / Troubleshooting

**`error: environment variable GROQ_API_KEY is not set`**
Run `export GROQ_API_KEY=your_key` before running the CLI, or add it to your shell profile.

**`error: reading profile.md … run 'newsletter profile setup' first`**
You haven't created a profile yet. Run `newsletter profile setup`.

**`No articles matched your profile`**
Your `similarity_threshold` may be too high. Try lowering it in `newsletter.yaml` (e.g. from `0.72` to `0.60`). Also check that your `recency_days` window is wide enough.

**The same articles appear every run**
The embedding cache prevents duplicate API calls but not duplicate articles. To regenerate from scratch, delete `.cache/embeddings.json`.

**`newsletter generate` is slow on first run**
The first run embeds every article (one API call each). Subsequent runs reuse the cache and are much faster.

**HTML output looks unstyled**
Make sure the theme CSS file exists at `templates/themes/{theme}.css`. Run `newsletter profile show` to check which theme is configured.

**I want to add more RSS sources**
Edit `config/newsletter.yaml` and add URLs under `sources.rss`. Any RSS 2.0 or Atom feed works.
